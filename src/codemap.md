# src/

## Responsibility

Application source root for Health Made Clear Next.js 16 bilingual (EN/ES) health education site.

## Design Patterns

- App Router with static generation via `generateStaticParams`
- next-intl for i18n routing, message loading, and navigation helpers
- Client components hold user progress in React context (AppProviders) synced to localStorage
- Supabase for optional authenticated persistence (mock client falls back in dev)
- Content pipeline: MDX source -> build scripts -> bundled TS data -> server/client loaders

## Key Directories

| Directory     | Responsibility                                                                                  |
| ------------- | ----------------------------------------------------------------------------------------------- |
| `app/`        | Next.js App Router pages, layouts, API routes, middleware                                       |
| `components/` | React component tree (layout, learn, quiz, articles, dashboard, UI primitives, providers)       |
| `hooks/`      | Custom React hooks (auth, progress, scroll spy, focus trap)                                     |
| `i18n/`       | next-intl routing config and request handler                                                    |
| `lib/`        | Business logic: content loading, auth, Supabase, i18n utilities, preferences, analytics         |
| `data/`       | Bundled content arrays generated by build scripts (lessons, paths, quizzes, articles, glossary) |
| `types/`      | TypeScript type definitions for all content entities and database schema                        |
| `messages/`   | next-intl JSON message catalogs (en.json, es.json)                                              |

## Entry Points

- `middleware.ts`: i18n routing + Supabase session refresh + dashboard auth guard
- `app/[locale]/layout.tsx`: Root locale layout with providers, header, footer
- `app/[locale]/page.tsx`: Home page
