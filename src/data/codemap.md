# src/data/

## Responsibility

Bundled content data arrays — generated by build scripts from MDX source, consumed by loaders.

## Key Files

- `lessonBundles.ts`, `lessonBundles.en.ts`, `lessonBundles.es.ts`: Lesson arrays keyed by locale
- `pathBundles.ts`, `pathBundles.en.ts`, `pathBundles.es.ts`: Learning path arrays
- `quizBundles.ts`, `quizBundles.en.ts`, `quizBundles.es.ts`: Quiz arrays
- `articleBundles.ts`, `articleBundles.en.ts`, `articleBundles.es.ts`: Article arrays
- `glossaryBundles.ts`, `glossaryBundles.en.ts`, `glossaryBundles.es.ts`: Glossary term arrays
- `glossary.ts`: Glossary lookup utilities
- `lessons.ts`, `articles.ts`, `paths.ts`: Locale-agnostic content arrays
- `learningPaths.ts`: Learning path metadata
- `searchIndex.en.ts`, `searchIndex.es.ts`: Client-side search index
- `articleBundles.ts`: Re-exports locale-specific article bundles

## Data Flow

1. MDX files in `content/` are parsed by scripts (`scripts/bundle-*.ts`)
2. Build outputs are TypeScript arrays exported from these files
3. Loaders in `src/lib/` import these bundles for server/client consumption

## Integration

- Consumed by: `src/lib/lessons/loadLessons.ts`, `src/lib/paths/loadPaths.ts`, etc.
- Generated by: `scripts/bundle-lessons.ts`, `scripts/bundle-paths.ts`, etc.
