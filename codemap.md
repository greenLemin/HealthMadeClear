# Repository Atlas: Health Made Clear

## Project Responsibility

A bilingual (EN/ES) Next.js 16 static health education site. Teaches health literacy through lessons, learning paths, quizzes, articles, and a glossary. Optional Supabase auth for progress persistence. No server database required — works fully client-side with localStorage.

## System Entry Points

- `src/middleware.ts`: i18n routing + Supabase session + dashboard auth guard
- `src/app/[locale]/layout.tsx`: App shell — providers, header, footer, SEO metadata
- `src/app/[locale]/page.tsx`: Home page with hero, featured lessons, learning paths
- `next.config.mjs`: Next.js config with i18n plugin, CSP headers, Supabase env bridging
- `package.json`: Scripts for dev, build, test, content bundling

## Core Architecture

```
content/*.mdx → scripts/bundle-*.ts → src/data/Bundles.*.ts → src/lib/*/load*.ts → app/[locale]/*/page.tsx
```

- **SSG**: All pages statically generated per locale
- **State**: React context (AppProviders) synced to localStorage for anonymous progress
- **Auth**: Optional Supabase with mock client fallback in dev
- **i18n**: next-intl with `[locale]` path prefix, EN/ES message catalogs

## Key Design Decisions

- **No server database required**: Core functionality works with localStorage. Supabase is optional for cross-device sync.
- **Content as code**: MDX source -> bundled TS -> runtime. Enables type safety, code splitting, and static generation.
- **Mock client**: Local dev works without real Supabase credentials via in-memory mock.
- **Pref bootstrap**: Inline script before hydration reads cookies to prevent flash of unstyled content.
- **CSP headers**: Strict content security policy configured in next.config.mjs.
